2. API SERVER — Express 5  |  Node.js  |  TypeScript  |  Port 8080  |  Prefix: /api

CURRENT ENDPOINTS DONE

GET  /api/healthz
  → Health check

GET  /api/speech/token
  → Azure auth token for client SDK

POST /api/speech/stt
  → Speech-to-Text proxy
  → Accepts: webm, mp4, wav (any audio)
  → ffmpeg converts to 16kHz WAV PCM
  → Forwards to Azure STT API
  → Returns: recognized text + confidence

POST /api/speech/tts
  → Text-to-Speech proxy
  → Input: { text, voice, rate }
  → Generates SSML with prosody rate
  → Returns: audio/mpeg stream
  → 4 Azure Neural voices (ID + EN)

5. KEY DATA FLOWS

READING FLOW (Core Experience)

 1. Student opens book from Explorer/Collection
 2. App calls GET /api/books/:id
 3. Server fetches digitized text from Blob Storage
 4. Returns page content to client
 5. Student taps Play (or voice command: "play")
 6. App sends page text to POST /api/speech/tts
 7. Server generates SSML → Azure Neural TTS
 8. Azure returns MP3 → streamed to client
 9. Audio plays via HTMLAudioElement or expo-av
10. On page finish → auto-advance to next page
11. Progress saved via POST /books/:id/progress

B2B INSTITUTION & PUBLISHER FLOW

Institution Flow:
 1. Institution signs agreement (offline)
 2. Admin creates business account in web dashboard
 3. Admin assigns book stock to institution
 4. Admin generates redeem tokens (codes)
 5. Institution distributes codes to students
 6. Student opens app → Redeem Token screen
 7. Enters code → POST /api/redeem
 8. Server validates & assigns book access

Publisher Flow:
 1. Publisher uploads EPUB/PDF via admin web
 2. System digitizes: extract text + metadata
 3. Admin QA checks → toggle Public when ready
 4. Publisher earns margin% from subscriptions
