🔧 REPLIT PROMPT: Fix Missing API Endpoint in Production Build
Problem Statement
Voice commands (TTS/STT) dan Book Reading tidak berfungsi di APK production build, padahal berfungsi di Replit dev mode.

Root Cause Identified
File: artifacts/mobile/services/speech.ts Line 3

const API_BASE = Platform.OS === "web"
  ? "/api"
  : `https://${process.env.EXPO_PUBLIC_DOMAIN}/api`;
Issue:

✅ Di Replit dev: EXPO_PUBLIC_DOMAIN di-set via npm script
❌ Di APK production: EXPO_PUBLIC_DOMAIN = undefined
❌ API endpoint jadi: https://undefined/api → Request gagal
❌ Semua TTS, STT, dan book reading fail silently
Current State
package.json (dev only):

{
  "scripts": {
    "dev": "EXPO_PUBLIC_DOMAIN=$REPLIT_DEV_DOMAIN ..."
  }
}
eas.json (production - MISSING env vars):

{
  "build": {
    "preview": {
      "android": { "buildType": "apk" }
      // ❌ NO env variables set
    },
    "production": {
      "android": { "buildType": "app-bundle" }
      // ❌ NO env variables set
    }
  }
}
Solution
Step 1: Identify Production API Domain
Pertanyaan untuk Anda:

Literaku API server di-deploy di mana?

[ ] Replit: https://<username>-<project-name>.replit.dev
[ ] Azure App Service: https://literaku.azurewebsites.net
[ ] Custom Domain: https://api.literaku.com
[ ] Other: ___________
Action: Cek di Replit:

Buka Project Literaku → API Server
Lihat URL yang ditampilkan (biasanya di banner atas)
Contoh: https://Your-Username-literaku.replit.dev
Step 2: Update eas.json Dengan Environment Variables
File: artifacts/mobile/eas.json

Replace entire file with:

{
  "cli": {
    "version": ">= 16.0.0"
  },
  "build": {
    "development": {
      "developmentClient": true,
      "distribution": "internal",
      "env": {
        "EXPO_PUBLIC_DOMAIN": "REPLACE_WITH_REPLIT_DEV_DOMAIN"
      },
      "android": {
        "buildType": "apk",
        "gradleCommand": ":app:assembleDebug"
      }
    },
    "preview": {
      "distribution": "internal",
      "env": {
        "EXPO_PUBLIC_DOMAIN": "REPLACE_WITH_PRODUCTION_API_DOMAIN"
      },
      "android": {
        "buildType": "apk"
      }
    },
    "production": {
      "env": {
        "EXPO_PUBLIC_DOMAIN": "REPLACE_WITH_PRODUCTION_API_DOMAIN"
      },
      "android": {
        "buildType": "app-bundle"
      }
    }
  }
}
Step 3: Replace Placeholder Domains
Ganti placeholder dengan nilai sebenarnya:

Placeholder	Ganti Dengan	Contoh
REPLACE_WITH_REPLIT_DEV_DOMAIN	Domain dev Replit (tanpa https://)	Your-Username-literaku.replit.dev
REPLACE_WITH_PRODUCTION_API_DOMAIN	Domain production API (tanpa https://)	Your-Username-literaku.replit.dev atau domain custom
Contoh Final (jika pakai Replit):

{
  "cli": {
    "version": ">= 16.0.0"
  },
  "build": {
    "development": {
      "developmentClient": true,
      "distribution": "internal",
      "env": {
        "EXPO_PUBLIC_DOMAIN": "Your-Username-literaku.replit.dev"
      },
      "android": {
        "buildType": "apk",
        "gradleCommand": ":app:assembleDebug"
      }
    },
    "preview": {
      "distribution": "internal",
      "env": {
        "EXPO_PUBLIC_DOMAIN": "Your-Username-literaku.replit.dev"
      },
      "android": {
        "buildType": "apk"
      }
    },
    "production": {
      "env": {
        "EXPO_PUBLIC_DOMAIN": "Your-Username-literaku.replit.dev"
      },
      "android": {
        "buildType": "app-bundle"
      }
    }
  }
}

Why This Fixes It
Before:

APK Build → EXPO_PUBLIC_DOMAIN=undefined
→ API_BASE = "https://undefined/api"
→ Fetch fails → No sound
After:

APK Build → EXPO_PUBLIC_DOMAIN="your-api-domain.replit.dev"
→ API_BASE = "https://your-api-domain.replit.dev/api"
→ Fetch successful → TTS works ✓
