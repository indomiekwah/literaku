LITERAKU — System Design

Voice-First Accessible Reading Platform for Visually Impaired Students • EN/ID Bilingual • Azure-Powered
📱 Client Layer
📱

Mobile App (Literaku)

Expo • React Native • TypeScript
  • Voice-first navigation via STT + CLU intent matching
  • TTS narration with queued playback & word highlighting
  • WCAG AAA: 48-78px touch targets, 18-28px bold text
  • TalkBack/VoiceOver support, SwipeHintBar + mic button
  • Screen flow: Splash → Login → Home → Explorer/Library/Reader
  • Book reader: section-based, auto-scroll during TTS
  • Settings: voice, speed, language (EN/ID), text size
  • AI-powered book summarization
Voice Flow: User speaks → expo-av records (.m4a) → API /speech/stt → transcript → voiceRouter regex match → fallback Azure CLU → intent (e.g. nav_home) → router.push()
🖥️

Admin Dashboard

React 19 • Vite 7 • Tailwind • shadcn/ui
  • Super Admin:
  • Dashboard stats & global analytics (Recharts)
  • Institution management (CRUD)
  • Operator account management
  • Book catalog management
  • Operator:
  • Student directory & registration (CSV bulk)
  • Book assignment to students
  • Per-student reading progress reports
  • Digitization request submission
Auth: JWT login (email/password) → localStorage → Bearer header → role-based routing
Mobile (HTTPS) ⇣ REST API + FormData (audio) ⇣ Dashboard (HTTPS)
